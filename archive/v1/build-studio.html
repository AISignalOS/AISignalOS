Every asset moves through five stages. Use this as a mental map for what to build next.

1

Input

A raw, repeated task — the messy thing you keep doing by hand.

2

System

The task written down as clear, repeatable steps anyone could follow.

3

Asset

The system distilled into a reusable prompt, skill, MCP server, or template.

4

Tool

The asset wrapped in an interface so anyone can run it without the internals.

5

Product

A tool that solves an expensive, repeated problem — packaged, priced, shipped.
